## Notes / followups

- **sqlc-sqlite limitation:** The CTE-chained `WITH ... INSERT ... SELECT`
  parses fine in sqlite proper but trips sqlc's parser with
  `relation "commits_agg" does not exist`. The query lives in Go as a
  `const aggregateSQL` and is run with positional `?1..?4` parameters
  inside the per-repo tx, alongside the sqlc-typed DELETE. 0034–0036
  may hit the same wall depending on their SQL shape — if so, follow
  the same pattern.
- **Commits additions/deletions stay zero.** The current commits ingest
  writes `Additions: 0, Deletions: 0` (`internal/ingest/commits/runner.go:186`),
  so the lines-changed columns are sourced from PRs on merge day only.
  If we ever wire up per-commit diffstat we can switch the source — no
  schema change needed.
- **0037 hook is free.** Idempotency lives in `aggregateRepo` (DELETE +
  INSERT in one tx), so a manual range re-run is just a loop over
  `Aggregate(ctx, date)` per local date.
