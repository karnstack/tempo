# 0039 — /api/v1/tokens CRUD with encrypted PAT

## What changed

- `internal/secret/secret.go` (new) — `Box` wrapping a 32-byte AES-256-GCM AEAD; `NewBox(key)`, `(*Box).Encrypt`/`Decrypt`. Wire format: `nonce ‖ ciphertext`. Random nonce per encrypt.
- `internal/secret/fx.go` (new) — `NewBoxFx(*config.Config)` adapter.
- `internal/secret/secret_test.go` (new) — 8 tests: round-trip, distinct nonces, tamper, wrong-key, too-short, empty plaintext, key-length validation (short + long).
- `internal/storage/sqlite/queries/connections.sql` — added `CountConnectionsByToken :one`.
- `internal/storage/sqlite/sqlitedb/connections.sql.go`, `querier.go` — regenerated by `make sqlc-generate`.
- `internal/api/tokens/tokens.go` (new) — `Configure(e, l, m, q, box)` mounts `GET/POST /api/v1/tokens` and `DELETE /api/v1/tokens/:id` behind `RequireSession`. `TokenDTO{id,label,scopes,expires_at,created_at}` (no PAT exposure ever). `tenantIDFromSession` helper looks up the user once per request to scope queries. POST validates non-empty label + PAT (post-trim). DELETE returns 409 with `{error,connection_count}` if connections reference the token.
- `internal/api/tokens/tokens_test.go` (new) — 13 integration tests against real SQLite + migrations: POST happy/trim/empty-label/empty-pat/bad-json/no-cookie/with-expires; GET multi/no-cookie; DELETE happy/missing/other-tenant/in-use/no-cookie. Happy path verifies the encrypt round-trip by reading the row back and decrypting via the same `Box`.
- `internal/api/run.go` — `Run` and `configureRoutes` take `*secret.Box`; calls `tokens.Configure(e, l, m, q, box)`.
- `cmd/tempo/main.go` — provides `secret.NewBoxFx` in the fx graph.

## Followups

- `tenantIDFromSession` is duplicated in `me` (inline) and `tokens` (helper). When 0040 (connections) lands, hoist into `web.CurrentUser(ctx, q) (*sqlitedb.User, error)` — three call sites is the threshold for the abstraction.
- Ingest worker (0026) reuses `*secret.Box` to decrypt `encrypted_pat` per token before instantiating each `github.Client`. The fx provider is already in place; 0026 just needs to depend on `*secret.Box`.
- No PAT preview on list — if user feedback reveals confusion between same-labelled tokens, add a `last4` column populated at create time (cheaper than decrypt-on-list, no metadata exposure beyond what the user already revealed).
