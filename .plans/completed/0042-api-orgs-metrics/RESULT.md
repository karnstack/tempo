## Notes / followups

- **`CAST(SUM(...) AS INTEGER)` is the antidote** to the
  sqlc-sqlite "aggregate returns interface{}/string" trap that
  burned 0036 with `MIN(timestamp)`. Forced int64 scan targets,
  no interface{} mess.
- **No percentile aggregation.** The response intentionally omits
  `lead_time_seconds_p50/p90`, `time_to_first_review_seconds_*`,
  and `response_minutes_p50`. Pooling per-repo percentiles without
  raw samples produces statistically wrong numbers; for fleet-wide
  percentiles, future work could pull from the raw event tables.
- **Date parsing duplicated** for the third time (tokens uses none,
  repos has it, orgs now repeats). 0043 will be #3 of the parser
  duplications; that's when an `internal/api/web/daterange.go`
  extraction makes sense.
- **`tenantIDFromSession` is now duplicated four times** (tokens,
  connections, repos, orgs). Same extraction trigger as above.
- **Cross-tenant repos under the same owner are isolated by the
  WHERE r.tenant_id = ?** clause in the SUM queries plus the
  in-Go `r.Owner != org` filter on `ListReposByTenant`. The 404
  for "no repos under this org" doubles as the cross-tenant
  isolation check.
