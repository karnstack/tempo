# 0035 — Cycle time + lead time rollup p50/p90

## Files changed

- `internal/storage/sqlite/queries/daily_repo_stats.sql` — added
  `UpsertRepoLeadTime :exec`. Writes only the two lead-time columns;
  count columns rely on schema DEFAULT 0 on INSERT and the
  ON CONFLICT DO UPDATE clause deliberately omits them so 0034's
  count writes are preserved.
- `internal/storage/sqlite/sqlitedb/daily_repo_stats.sql.go`,
  `internal/storage/sqlite/sqlitedb/querier.go` — regenerated by sqlc.
- `internal/rollup/cycletime/aggregator.go` — `Aggregator` type with
  `Name()` / `Aggregate()`. Fetches merged PRs in
  `[date, date+24h)` per repo via the existing
  `ListMergedPullRequestsByRepoBetween`, computes integer-second
  durations in Go, sorts, and writes nearest-rank p50/p90 via
  `UpsertRepoLeadTime`. Zero merged PRs → `*int64(nil)` → SQL NULL,
  so a previous-run-with-data → next-run-without-data transition
  clears stale percentiles.
- `internal/rollup/cycletime/module.go` — fx module providing
  `*Aggregator` into the `"rollup.aggregators"` value group.
- `internal/rollup/cycletime/aggregator_test.go` — 9 tests covering
  empty DB, single PR, nearest-rank math (4 PRs ⇒ p50=sorted[1],
  p90=sorted[3]), merged_at window boundaries, multi-repo
  partitioning, archived-repo skip, percentile-clear on empty rerun
  with counts intact, disjoint-columns contract (count columns
  preserved when aggregator runs after a UpsertDailyRepoStats seed),
  and negative-duration filtering.
- `cmd/tempo/main.go` — `cycletime.Module` registered alongside the
  ingest modules, `engineerstats.Module`, and `repostats.Module`.

## Notes / followups

- **Em-dash in SQL comments breaks sqlc.** The sqlc parser choked on
  `—` in a comment with `no viable alternative at input` until I
  swapped to `;`. Worth keeping all SQL comments ASCII-only —
  apologies for relearning the same lesson next aggregator.
- **`ListMergedPullRequestsByRepoBetween` infers `*time.Time` params.**
  sqlc inherits nullability from the column on the comparison RHS;
  since `merged_at` is nullable, `FromTs` / `ToTs` are `*time.Time`,
  so the aggregator passes `&from` / `&to`. If we ever need this
  query with non-nullable param types, add an explicit `sqlc.arg`
  with a cast, or split into a dedicated query.
- **Nearest-rank percentile.** `idx = ceil(q*n/100) - 1`, clamped.
  Chosen over linear interpolation for stable integer-second values
  and easy-to-spell-out tests. Dashboard rendering can switch to a
  histogram later without a schema change.
- **Negative durations filtered.** The PR ingest hands us GitHub
  timestamps directly; clock-skew rows shouldn't happen but the
  filter is cheap insurance.
- **0037 hook is free.** Same shape as 0033/0034 — re-running a date
  is `Aggregate(ctx, date)`, idempotent by UPSERT including the
  no-data → NULL transition.
- **Disjoint-columns contract, both directions.** 0034's
  `RESULT.md` enshrined it from the counts side; this task closes
  the loop from the percentiles side with
  `TestAggregate_CountColumnsPreserved`.
