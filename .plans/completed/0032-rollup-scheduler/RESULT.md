# 0032 — Rollup scheduler (daily 02:00 instance-local)

## What changed

- `migrations/0003_daily_rollups.sql` — added `rollup_runs(date, kind,
  started_at, finished_at, ok, error)` table in place (PK
  `(date, kind)`, index on `date`).
- `internal/storage/sqlite/queries/rollup_runs.sql` — new file with
  `UpsertRollupRunStart`, `FinishRollupRun`, `GetRollupRun`,
  `ListSuccessfulRollupDates`.
- `internal/storage/sqlite/sqlitedb/rollup_runs.sql.go` + `models.go`
  + `querier.go` — regenerated by sqlc.
- `internal/rollup/aggregator.go` — new `Aggregator` interface +
  `NoopAggregator` sentinel.
- `internal/rollup/scheduler.go` — new `Scheduler` with `Loop`,
  `Tick`, `CatchUp`, `RunDate`, plus pure helpers
  (`nextFire`/`pastFireTime`/`bucketDate`/`localMidnight`) and a
  package-local `closingCtx` mirroring the ingest pattern.
- `internal/rollup/scheduler_test.go` — 11 hermetic tests (pure
  helpers, RunDate, Tick, CatchUp, Loop). Uses an atomic-counter
  fakeAgg so the goroutine-driven Loop test is race-clean.
- `internal/rollup/export_test.go` — test-only export of
  `BucketDate` and `NextFire` so the external `_test` package can
  drive the pure helpers without changing the production surface.
- `internal/rollup/run.go` — fx `RunParams` + `Run` mirroring
  `ingest.Run`. Pulls aggregators from the `"rollup.aggregators"`
  value group.
- `cmd/tempo/main.go` — `fx.Invoke(rollup.Run)` next to
  `fx.Invoke(ingest.Run)`. Boot log shows hour, tz, aggregator count.

No new migration file (per pre-launch edit-in-place memory). No
config changes (`Rollup.Hour` + `Rollup.Timezone` already shipped).
