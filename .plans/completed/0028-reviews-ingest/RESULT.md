# Result — 0028 Reviews/comments ingest

## What changed

- **Schema (`migrations/0002_raw_events.sql`, edit-in-place pre-launch).**
  Added `updated_at TIMESTAMP NOT NULL` column to `pull_requests` between
  `created_at` and `merged_at`, plus index `pull_requests_repo_updated_idx`
  on `(repo_id, updated_at)`. Drives the incremental "which PRs need their
  sub-resources refreshed" filter for prconvo (and is intrinsically useful
  for the upcoming rollups in 0033–0036).

- **`internal/storage/sqlite/queries/pull_requests.sql`.** Extended
  `UpsertPullRequest` with `@updated_at` (insert + conflict). Added new
  query `ListPullRequestsByRepoUpdatedSince(repo_id, since)` returning PRs
  with `updated_at > @since` ordered ASC.

- **`internal/storage/sqlite/sqlitedb/*.go`.** Regenerated by sqlc.

- **`internal/ingest/prs/runner.go`.** Now passes `UpdatedAt: pr.UpdatedAt`
  through to `UpsertPullRequest`. Existing prs tests still pass (the
  cassettes already carried `updatedAt`).

- **`internal/github/prconvo/fetcher.go`.** Exported the three GraphQL
  query strings as `ReviewsQuery`, `ReviewCommentsQuery`,
  `IssueCommentsQuery` so the ingest runner's cassette gen test can build
  matching request bodies without duplicating them.

- **`internal/ingest/prconvo/`** (new package). Per-PR ingest runner:
  - `doc.go` — package overview + cursor convention.
  - `run.go` — fx `Module` providing `*Runner` into the
    `"ingest.runners"` value group (mirrors `prs.Module`).
  - `runner.go` — `Runner` with `*sqlitedb.Queries` + `*zap.Logger` +
    optional clock. `Name() = "prconvo"`. `Run` walks the connection's
    non-archived repos, loads cursor `prconvo:<owner>/<name>`, lists PRs
    whose `updated_at > cursor`, then for each PR pages reviews,
    review-thread comments, and issue (conversation) comments — upserts
    authors into `gh_users` (Ghost → 0), upserts rows, and advances the
    per-repo cursor to `max(pr.updated_at)` of successfully processed PRs.
    Truncated review threads log a `zap.Warn` but don't fail. PENDING
    reviews (`submitted_at == nil`) are dropped (the column is NOT NULL).
  - `runner_test.go` — 6 hermetic tests, VCR replay against committed
    cassettes:
      - happy path: 1 PR, 2 reviews (1 user + 1 ghost), 2 review comments
        (1 user + 1 ghost), 1 issue comment;
      - multi-PR cursor advancement;
      - truncated thread warning (no failure);
      - existing cursor filters older PRs (only newer PR queried);
      - empty cases: no repos, no updated PRs (zero HTTP, no cursor
        write);
      - per-PR failure halts the repo and leaves the cursor unwritten.
  - `cassettes_gen_test.go` — `//go:build gen` cassette author; mirrors
    the prconvo fetcher's pattern.
  - `testdata/*.json` — 5 committed cassettes.

- **`cmd/tempo/main.go`.** Added `prconvo.Module` to the fx graph.
  Scheduler boot log now reports `runners=2`.

## Follow-ups

- **`gh_users.last_seen_at` granularity.** Reviewers/commenters currently
  inherit the parent PR's `UpdatedAt` rather than the review/comment's
  own timestamp. Acceptable for v1 (the field is monotonic-enough). If
  later rollups want per-actor freshness, plumb the individual
  `submitted_at`/`created_at` through.
- **Per-PR pagination on sub-resources.** All three loops handle
  `HasNext`, but no test currently exercises a multi-page reviews response.
  Worth adding once we see a PR with >100 reviews in the wild (rare).
- **Empty-PR skip optimization.** v1 makes 3 GraphQL calls per "updated"
  PR even when reviews/comments are empty. Could collapse to 1 call by
  inlining `totalCount` and skipping zero-count connections. Within budget
  (5000 points/hour) at current scale.
